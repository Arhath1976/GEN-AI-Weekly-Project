---

## 🔗 LangChain

LangChain is a framework used to build applications powered by Large Language Models (LLMs).

It helps in:
- Connecting LLMs with external data  
- Managing prompts and workflows  
- Building intelligent pipelines  

---

## 📥 Data Ingestion

Data ingestion is the process of collecting and loading data into the system.

### Sources Used:
- Text files  
- Documents  
- Custom datasets  

👉 This allows AI to work with **real-world data instead of only pre-trained knowledge**

---

## ✂️ Text Splitting

Large documents are divided into smaller chunks for better processing.

### Why it is important:
- Improves retrieval accuracy  
- Reduces token usage  
- Enables efficient search  

---

## 🧠 Knowledge Graph

A Knowledge Graph represents data in a structured format using:

- Nodes (entities)  
- Edges (relationships)  

### Benefits:
- Better context understanding  
- Improved reasoning  
- Structured data representation  

---

## 🤖 Retrieval-Augmented Generation (RAG)

RAG is a technique where AI:

1. Retrieves relevant data from a database  
2. Uses that data to generate accurate responses  

👉 Combines:
- Information retrieval  
- Language generation  

### Advantages:
- More accurate answers  
- Reduced hallucinations  
- Context-aware responses  

---

## 💬 Simple RAG Chatbot

A chatbot was created using RAG that:

- Takes user queries  
- Searches relevant data  
- Generates responses based on retrieved information  

### Features:
- Context-based answering  
- Improved accuracy  
- Works with custom data  

---

## ⚙️ Workflow

1. Data ingestion  
2. Text splitting  
3. Store embeddings / knowledge  
4. Retrieve relevant chunks  
5. Generate response using LLM
